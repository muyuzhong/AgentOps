# Phase 3.5 Eval Spike And Minimal Stop-Hook Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Answer two architectural assumptions before Phase 4 is designed, and harden the declaration chain that the whole evaluation depends on:

1. How far can purely deterministic rules go in evaluating **session quality**, using **scope drift** as the single probe dimension?
2. Does the **declaration vs ground truth** reconciliation chain actually work end to end — taking an agent's `TaskReport` (declaration) and a `DiffSummary` (truth) and turning the gap into a useful signal?

Plus a real, kept prerequisite: a **minimal stop-hook** so an agent that forgets to append its task report is reminded. Without it, the agent can skip the declaration and the entire reconciliation chain breaks.

**Spike discipline:** Scope drift is the ONLY evaluation dimension in this phase. Rules are deterministic only. The probe may hardcode, cut corners, and be thrown away. Its primary deliverable is **answers** — a findings document marking exactly where deterministic rules suffice and where an LLM becomes necessary — not production-grade evaluation code. The reconciliation core graduates to Phase 4 only if it proves out.

**Architecture:** Two parts, in order.

```text
Part A (kept):      session-log freshness check -> agentops check-session-log -> Stop-hook reminder
Part B (throwaway): TaskReport (declaration) + DiffSummary (truth) -> reconcile -> scope-drift findings -> answers doc
```

**Tech Stack:** Python 3.11+, standard library (`dataclasses`, `enum`, `hashlib`, `json`, `pathlib`, `argparse`), `pytest`. Reuse existing `TaskReport`, `SessionTrace`, `DiffSummary`, `ChangedFile`, `TranscriptParser`, `GitAnalyzer`, `DiffParser`. No new third-party dependency.

## Scope Guard

Implement only the minimal stop-hook and a single-dimension, deterministic scope-drift probe.

Do not add:

- a finished `agentops eval` command or the full Phase 4 evaluation workflow;
- any second evaluation dimension (context quality, verification sufficiency, etc.);
- any LLM call — mark insertion points in comments only;
- a numeric session-quality score — emit structured scope-drift findings, not a grade;
- memory store, historical trend storage, or a watcher process;
- automatic edits to application source code (the hook may write only its own state file and `.claude/settings.json` managed block);
- full-transcript ingestion (continue to consume the bounded `SessionTrace`).

`agentops scan` and `agentops init` behavior must remain unchanged except for the optional, additive hook registration in Task 3.

## Contracts

### Part A — Session-Log Freshness (kept)

```python
@dataclass(frozen=True)
class SessionLogState:
    byte_size: int
    sha256: str
    task_count: int

    def to_dict(self) -> dict[str, object]: ...


@dataclass(frozen=True)
class SessionLogCheck:
    has_new_content: bool
    previous: SessionLogState | None
    current: SessionLogState
    reminder: str | None   # set only when has_new_content is False
```

- State persists at `.agentops/.session-log-state.json` (managed, gitignore-friendly).
- "New content" is decided by a larger `byte_size` AND a changed `sha256` relative to the recorded baseline; a missing baseline counts as "no new content" and emits the reminder while recording the current state.
- `task_count` reuses `TranscriptParser` and is informational; never open raw transcripts.
- The check is deterministic and side-effecting only on its own state file.

### Part B — Scope-Drift Reconciliation (spike)

```python
@dataclass(frozen=True)
class ScopeDriftFinding:
    code: str            # e.g. "undeclared_change", "declared_not_changed", "cross_module_breadth"
    evidence: tuple[str, ...]
    llm_needed: bool     # True where a deterministic rule cannot judge intent


@dataclass(frozen=True)
class ScopeDriftReport:
    declared_paths: tuple[str, ...]
    changed_paths: tuple[str, ...]
    findings: tuple[ScopeDriftFinding, ...]
```

- Input is one `TaskReport` (declaration: `context_used` + `changes` path mentions) and one `DiffSummary` (truth).
- Deterministic rules: files changed but never declared; files declared but unchanged; number of distinct top-level modules touched vs a claim of a narrow change.
- Every place where judging "is this change within the task's intent?" requires semantics MUST set `llm_needed=True` and carry a `# SPIKE: LLM insertion point` comment. Do not call an LLM.

## Exit Criteria

Phase 3.5 is complete when:

- `agentops check-session-log` reliably reminds when no task report was appended and stays quiet when one was;
- the existing `agentops scan` / `agentops init` behavior is unchanged (plus the optional additive hook block);
- `reconcile_scope` turns a `TaskReport` + `DiffSummary` into deterministic scope-drift findings, with LLM-needed gaps explicitly marked;
- the chain has been run on at least one real session and its output captured;
- the findings doc answers both assumptions and gives a concrete Phase 4 recommendation;
- `python -m pytest -v` passes.

Phase 4 is planned only after these answers exist — do not pre-design the Phase 4 evaluation architecture in this phase.
